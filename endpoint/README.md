# Endpoint

## Ownership

Endpoint agent, commissioning, adapter SDK, adapter and installer implementation files belong here when a scoped task approves them.

## Phase 1 status

This directory is intentionally a placeholder for Epic 1. It does not define or implement a Windows service, local API, adapter contract, installer, cloud client or runtime behaviour.

## Build commands

No endpoint build command exists yet. Future tasks must add build commands with the implementation they authorize.

## Test commands

No endpoint-specific test command exists yet. Repository documentation checks run from the root with:

```sh
npm run check:docs
```
