# Product Terminology

This document is the official product dictionary for the Endpoint Platform.

One word has one meaning. Do not redefine terms in later documents, code, contracts, tickets or user-facing copy. If a new term is needed, add it here in the same change that introduces the term.

## Rules

- Use the terms below exactly as defined.
- Prefer these terms over synonyms in code, APIs, documentation and operational procedures.
- Do not use reserved terms until an approved decision activates them.
- Do not expose runtime-specific terms in cloud APIs unless an approved contract explicitly requires them.

## Dictionary

| Term | Definition | Notes |
| --- | --- | --- |
| Platform | The complete Endpoint Platform ecosystem, including cloud, nodes, web applications and tooling. | Never refers to a single application. |
| Node | A managed endpoint device running the Node Agent. | May host one or more adapters. |
| Node Agent | The background application responsible for identity, communication, configuration, health and adapter management. | Never directly processes media. |
| Adapter | A software component that translates platform capabilities into runtime-specific operations. | TouchDesigner is the first adapter. |
| Runtime | A third-party application that performs media processing. | Examples include TouchDesigner, VLC, OBS and vMix. |
| Capability | A logical function exposed by an adapter. | Hardware-independent. |
| Command | A transient instruction sent to a node. | Not persistent configuration. |
| Desired State | The configuration the cloud wants the node to achieve. | Versioned. |
| Reported State | The node's current observed condition. | Generated by the node. |
| Configuration | Persistent settings stored by the platform. | Version controlled. |
| Preset | A named collection of configuration values. | User-facing. |
| Scene | A runtime-specific implementation detail. | Avoid using in cloud APIs. |
| Room | A logical AV environment managed by one or more nodes. | Not necessarily a physical room. |
| Asset | Media content such as images, videos or documents. | Managed separately from configuration. |
| Pairing | The secure process of assigning a node to a deployment. | One-time operation. |
| Deployment | An isolated customer environment created from the shared codebase. | Blue Elephant is the first deployment. |
| Tenant | Reserved term. Not used in Phase 1. | Future concept only. |
| Operator | End user with simplified control permissions. | Formerly "User". |
| Technician | User with operational configuration permissions. | Limited by Administrator policies. |
| Administrator | User with full platform configuration authority. | Highest role. |
| Room Code | Temporary code used to access the room control interface. | Authentication method, not identity. |
| Review Gate | Mandatory approval point between Epics. | Stops progression until approved. |
| Decision Request | Formal request for an unresolved material decision. | Never solved silently. |

## Change control

Terminology changes are product changes. A pull request that adds, renames or removes a term must update affected documentation, contracts, tests and code together.
