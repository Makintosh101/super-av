# Standards

These documents define the repository-wide standards for product language, naming, errors, versioning, APIs, code, data and documentation.

## Core standards

- `PRODUCT_TERMINOLOGY.md` — official product dictionary.
- `NAMING_STANDARDS.md` — naming rules for documents, identifiers, APIs, JSON, databases, code and configuration.
- `ERROR_STANDARD.md` — structured error format, severity values and reserved error-code ranges.
- `VERSIONING_POLICY.md` — versioning rules for platform components, APIs, contracts, configuration, migrations, releases and compatibility.
